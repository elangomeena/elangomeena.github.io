You've got your Python data — now let's serve it. In this post, you'll build a `GET` endpoint that returns JSON, a `POST` endpoint that validates input, and test everything through Swagger UI without touching a frontend.

## What You're Doing

→ Build a `GET` endpoint — serve your Python data as JSON
→ Build a `POST` endpoint — accept input, validate it, respond to it
→ Open Swagger UI — test everything without touching a frontend

## Setup

Install FastAPI and uvicorn:

```bash
pip install fastapi uvicorn
```

## The GET Endpoint

**GET** — Fetches data from the server. Here we return a list and a dict as JSON.

```python
from fastapi import FastAPI

app = FastAPI()

fruits = ["apple", "banana", "cherry"]
person = {"name": "Meena", "age": 25, "city": "Bangalore"}

@app.get("/data")
def get_data():
    return {"fruits": fruits, "person": person}
```

Hit `GET /data` and FastAPI automatically serializes your Python objects to JSON.

## The POST Endpoint

**POST** — Accepts data from the client. Pydantic models handle validation automatically.

```python
from pydantic import BaseModel

class Item(BaseModel):
    name: str
    age: int

@app.post("/person")
def create_person(item: Item):
    return {"message": f"Received {item.name}, age {item.age}"}
```

**Pydantic BaseModel** — Declare your expected fields and types. FastAPI rejects invalid input with a `422` error before your function even runs.

## Full app.py

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

fruits = ["apple", "banana", "cherry"]
person = {"name": "Meena", "age": 25, "city": "Bangalore"}

@app.get("/data")
def get_data():
    return {"fruits": fruits, "person": person}

class Item(BaseModel):
    name: str
    age: int

@app.post("/person")
def create_person(item: Item):
    return {"message": f"Received {item.name}, age {item.age}"}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="127.0.0.1", port=8080)
```

## Run & Test

```bash
python app.py
```

Then open **[http://localhost:8080/docs](http://localhost:8080/docs)** — Swagger UI is auto-generated by FastAPI. You can test both endpoints directly from the browser.

> No frontend, no Postman, no extra setup — Swagger UI is built in.

## Quick Recap

- `@app.get` — read data, returns JSON
- `@app.post` — accept input, Pydantic validates it
- `/docs` — free Swagger UI, always available in dev
